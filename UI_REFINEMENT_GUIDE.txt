FRAGILITYGRAPH — UI REFINEMENT & GAP ANALYSIS GUIDE
=================================================

This document outlines the remaining gaps and specific instructions for refining the FragilityGraph frontend.

1. GRAPH INTERACTION FIXES
--------------------------
- Remove Node Flips: Disable the 'rotateY' animation on node hover. It causes choppiness and interferes with readability.
- Node Sizing: Decrease default node sizes (Root: 36px, others: 18-40px based on score).
- Label Placement: Ensure labels (file names) are positioned safely UNDER the nodes (text-valign: bottom).
- Edge Percentages: Edge labels must remain large (14px+) on hover and have zero shadows for maximum clarity.

2. BUTTON & FEEDBACK LOGIC
--------------------------
- Analyze Button State: When clicked, the button should immediately change text to "Analysing..." to provide instant feedback while the backend processes.
- Status Badge: Ensure the "Analysing" top-badge syncs correctly with the button state.

3. ASSET & ICON REFINEMENT
--------------------------
- Transparency Fixes: Ensure icons utilise 'object-fit: contain' and 'background: transparent'. Check for white-border artifacts in 'icons/analyze.png' and 'icons/impact.png'.
- Image Placeholders: See the "Regeneration Guide" below for creating high-quality replacements.
- Empty States: If a file has 0 dependencies, do not leave the graph blank. Display a dedicated "No Dependents" image/illustration in the center of the graph card.
